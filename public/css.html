  • Css Box Model

      • Note: Every html element should be treated as a box on screen.
      setting elements height and width makes it obvious.
      We can also manually contain element in box using Div(Content Division Element)

    • Margin
      • This is applicable to outside the border.
        It separates the element from other elements on the page,
        controlling the distance between neighboring boxes
    • Border
      • This is applicable to thickness of element.
        Border is the line that wraps around the padding and content.
        It visually separates the element from its surroundings.
    • Padding
      • This is the space between an element's content and its border.
        It creates internal spacing, pushing the border away from the content

        • Applications

      • #sample {border: 10px solid black}

        • This means 10px to all 4 sides and
          make it solid and of black color
      • #sample2 {border: solid black}
        • Make border of black and solid for visual
      • #sample3 {border-width: 10px 2px 4px 6px}
        • Apply 10px to top, 2px to right,
          4px to bottom and 6px to left
      • #sample4 {border-width: 12px 8px}
        • Apply 12px to both top and bottom and
          8px to both right and left

  • Css Cascade

      • This is order of importance in which css is being apply to html.

    1. Position(This give priority to the last applied style)
      • p {color: red;
        color: blue; }

        [blue will override red because of the position.]

    2. Specificity(This is rank next to the position)
      • li {color: coral}
      • .first-class {color: red;}
      • li[draggable] {color: purple;}
      • #first-id {color: green}

        • If all these selectors are all apply to an element tag(li)
        The "id selector"(#first-id) will be apply because it
        will override the other selectors Because it is the most
        specific selector. follow by "attribute selector",
        "class selector", and "element selector".
        Note: Though some believe "attribute selector and
        class selector" are on the same level. But in most cases
        it do override class selector.

    3. Type(This is next after Specificity)
      1. <link rel="stylesheet" href="">
      2. <style></style>
      3. <h1 style="">hey</h1>

        4. External link is the last to be consider as it pave way for
        other type. internal is next to External css. The inline
        is the first and will override the rest type of css.

    4. importance(This is in the first level in the Cascade steps.)
      • p {color: red;}
      • p {color: green !important;}

        • In this instance, green will be apply because
        of the important notation. it will override any
        other styling available be it inline, id and so on.

  • Css Combining Selectors

      • This is the combining multiple css selectors in order to target specific element in html

    • Grouping

        This is done using comma to separate element

      • selector1, selector2 {color: coral;}
    • Parent child Selectors

        Using ">" to select child. This is one level deep.

      • selector1 > selector2 {color: gold;}
    • Descendant Combination

        This comprises ancestors and the descendant
        with a space separating them. This means our
        ancestors will apply the style to the descendant
        as long as it has ancestors that matches the selector.
        We can have many level deep.

      • selector selector {font-size: 5rem;}
    • Chaining Selectors

        This apply style to where all the selectors are true.
        This does not have space between them.
        eg <p id="paragraph" class="bold dope">Hey</p> => p#paragraph.bold.dope
        [Useful to when we have same style on element across code].
        making element more specific.
        In the case of element involvement(h1, p).
        We should always start with them for clarity.

      • selector1selector2 {color: blue;}
    • Combining Combiners

        This is the combination of different combine selectors
        This involves ancestors space then chaining method
        or group method to combine different combination.

      • selector selector2selector3 {color: red;}

  • Css Positioning

      • This is a way we can place element or object using css

    • Static Position

        This is the default position of element.
        No Properties like Top, right e.t.c
        are not applicable to this position

    • Relative Position

        This unlike static position is relative
        to its position. Properties like top,
        right e.t.c are applicable to this

    • Absolute Position

        This affect the flow of html element.
        This position is relative to its Parent
        element. Properties like top, left e.t.c
        are applicable to it. It can as well be
        control by z-index because it can appear
        on top of other element.

    • Fixed Position

        This like absolute position also affect flow
        of an html. It is fixed unlike other position.
        Properties like top, right e.t.c are applicable
        to it as well. This make element to remain fix
        when scrolling.

    • Sticky Position

        Similar to fixed position

  • Css Display

      • This is away we can align our element on web page
      There are many types of display but we will be touching
      only 4 types here. Note: Span element can be use to put
      content within element because it's natural display property
      is "Inline".Visualize all types of display here => Github

    • Block Display

        This is the default display of all element. It
        means, element take full width of the entire
        line contain. It force other element to appear
        next line. It height and width can be change

    • Inline Display

        Contents is the determinant of this type. It uses
        the height and width of content to determine its
        sizes. Is height and width cannot be change. Other
        element can be place beside it as it doesn't take
        entire width as block display. Its height and width
        can't be change. It's fix by the demand of its contents.

    • Inline-Block Display

        This comprises both the "Inline" and The "Block"
        display abilities. It takes full width and can
        extend to other line if needed. Both width and
        height can be change. It accept element to be
        place side by side.

    • None Display

        As the name implies, it makes element disappear
        useful for element we don't want to display.
        Useful to make element of TO-DO list to disappear

  • Css Responsiveness

      • This is a way we make our website responsive
      to different screen sizes.

    • Media Query

        This is basics of responsiveness in vanilla css before
        introduction of flex and grid. this involves using:

      • @media(max-width: 720px) { .body { background-color: green; } }
        • Which denotes: When screen size is at 720px or lesser, adjust to new settings. This target smaller screen.

    • Grid

        Grid allow a 2 or 3 dimensional layout of website. This involves setting main div display to grid. e.g .main { display: grid; grid-template-columns: 1fr 1fr; grid-template-rows: 100px 150px 150px gap: 20px } .first { grid-column: span 2; } .card { background-color: red; } which simply means, create 2-columns and 3-rows,
        make the first div occupy two columns. So the first
        div will occupy two columns and will be 100px. The rest
        will occupy one column and will be 150px each with a gap
        of 20px.

    • Flexbox

        Flex allow 1 dimensional layout of website. .container { display: flex; border: 20px solid red; } .card { background: brown; border: 20px solid seagreen; height: 150px; flex: 1 } .first { flex 2 } .second { flex: 0.5; } This simply means: We set the main div display to flex then we add border, we set each div to brown then its border to seagreen, we set the height and we make it to 1 ratio. we set first div to be twice the size of default which is 1. we set second div to be half of the default div. So with the ratio as width, it makes it responsive.

  • Responsiveness In Details

    • Media Query

        Previously, we discuss about max-width. In media query, we also have min-width. for example:

      • @media(min-width:720px){
        h1 {font-size: 2rem}
        }
        • This means, when the screen size is at 720px,
        that is minimum of 720px and above. Increase the
        font size. This target bigger screen.

        Combining Breakpoint.

        We can combine breakpoint to target specific size eg:

      • @media(min-width:720px) and (max-width: 900px){
        ...
        }
        • This means, we want the effect on screen of range between
        720px and 900px. So anything bigger than 720px and less
        than 900px.

      • media(max-width:720px) and (min-width: 900px){
        ...
        }
        • Flipping it around make the interpretation to change. This
        means, we want changes below 720px on small screen and on
        bigger screen should be greater than 900px and higher.

        Device

        We can also see device usage. eg

      • @media screen(orientation: landscape) {
        ....
        }
        • We don't have to put 'screen' as it'll be applicable to all screen by default.
        Alternative to screen is print. This allow us to use media query to
        target only when our website is printed and to give it different layout.
        so we can either target screen or print and we don't have to be explicit
        by writing both the screen and the print out.

  • Flex

    • This is easy and neat modern tools. Which is simply implemented by creating a div that encapsulate the rest div's. Simple illustration is:

      .container { display: flex; gap: 10px; } p { background-color: darkseagreen; }

    Previously, we learned that div occupy full width while span is an inline element. p tag is also block element. img tag is an inline but with flexbox, all of the default behavior are overlook. And the width of each of these element inside flexbox will be based on the content size. So it'll try and let us have all our content on one line. We can then use Gap property to separate them. Default setting of a container(div) is like a block element(100% full width). We can also set it as: 'inline-flex'. Which is similar to the inline we already treated. It'll make flexbox occupy the space needed. Which means other things can also go and occupy in that same line. So we can either set the main container(div) as either 'flex' or 'inline-flex'.

    • Flex direction

        The default settings of flex is row. We can also set flex direction property to 'column.' When at default that is 'row'. the main axis tends to be on an horizontal line while the cross-axis will be on vertical line. Which when 'flex-basis' property is apply, it increases the width. If the flex-direction is set to 'column' the main axis will now be vertical and the cross-axis will now be the horizontal line. Which means when 'flex-basis' property is apply, it will increase the height.

      • .main { display: inline-flex; background-color: darkblue; border: solid rebeccapurple 5pt; gap: 120px; color: yellowgreen; font-size: 1.1rem; flex-direction: column; } .red { background-color: red; } .green { background-color: green; } .brown { background-color: brown; } .main > * { flex-basis: 80pt; }

        • In the above example, since the 'flex-direction' is column, flex-basis will increase the height of each items. If it is at default of row. The width will increase instead. We used 'inline-flex' to make the full-width not at default of block instead it should take the width of the content.

    • Flex-layout

      Align, justify and wrap. Note: you have to differentiate whether properties is from parent(container) or the child (flex-items). For example, 'order' can only be set on the flex-items. Default order of html element is '0'. Also, we have 'flex-wrap' which is applicable to the parent (container). The default is 'nowrap' which makes our container not responsive as it'll cut-off some items. Which we can set to 'wrap' which make our items continue to other line when the first line can't contain them. For Demonstration click => Demo

      Parent Properties(Flex-Container)
      • Flex-direction
      • Flex-wrap

          This is useful when we run out of space on the horizontal. The default settings is set to 'nowrap'. Which we can set to 'wrap', which move all element that the first line can't contain onto the next line. We also have 'wrap-reverse' which make things wrap from bottom right to the top left corner. Instead of the normal version which is from top left to bottom right.

      • Justify-content

        This sets the distribution of our items along main-axis. Properties are: 'flex-start', 'flex-end', 'space-evenly', 'space-around', 'space-between' and 'center'.

      • Align-content

          This is similar to 'align-items but only works with 'wrap'. When we set our items to wrap and we want to specify how the content should align, then 'align-content' comes to play. For example, if we applied 'align-content' to center and we made our flexbox wrap, then when the width of our window reduces everything will be muzzle up together. If second line is needed they'll go to the second line and then it'll look the 'align-content' value to see how it should be space on the screen. When items is 'nowrap' and we set the 'align-content', it doesn't really do anything. But when we set items to 'wrap', and then there is not enough width to have everything on one line, that's when our 'align-content' comes into action. Properties are: 'baseline', 'flex-start', 'flex-end', 'stretch' and 'center'.

      • Align-items

          Unlike the 'justify-content' which is applicable along horizontal axis. 'align-items' distribute across 'cross axis'. For this properties to work, we have to set its height. eg: height: 50vh; That is the 'viewport height'. Properties involves: 'baseline', 'flex-start', 'flex-end', 'stretch' and 'center'. If wrap was enable, the behavior will be cringe which we'll expatiate when we discuss 'align content'. Best to keep it as 'nowrap'. We can reach individual property using 'align-self'

      • Gap, row-gap, column-gap

        This controls the space between flex items. It applies that spacing only between items not on the outer edges.

      • Flex-flow

        This is the shorthand for both 'flex-direction and 'flex-wrap'
